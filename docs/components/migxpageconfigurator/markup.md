# Разметка вёрстки

Разметка — основа всей работы с компонентом. Вы берёте **готовую** вёрстку и навешиваете на нужные элементы атрибуты `data-mpc-*`: помечаете, где секция, где редактируемое поле, где список, где вызов сниппета. Переписывать вёрстку не нужно — только пометить. По этой разметке нарезка раскладывает шаблон на файлы-секции и чанки, на месте полей ставит плейсхолдеры Fenom, а контент переносит в админку.

::: tip
Скопируйте в проект файл `assets/components/migxpageconfigurator/css/mpc.css` — тогда IDE будет подсказывать доступные атрибуты прямо во время разметки.
:::

::: danger
У всех атрибутов префикс `data-mpc-`.
:::

## Заголовок шаблона

Файл шаблона должен начинаться со специального комментария — из него компонент создаёт MODX-шаблон и типовой ресурс:

```html
<!--##{"templatename":"Главная","pagetitle":"Главная страница","icon":"icon-home"}##-->
```

- `templatename` — имя будущего MODX-шаблона;
- `pagetitle` — заголовок создаваемого ресурса;
- `icon` — иконка ресурса (необязательно).

::: danger
Формат `<!--## … ##-->` менять нельзя — он распознаётся регулярным выражением, а содержимое внутри должно быть **валидным JSON**.
:::

## Секция

Секция — это смысловой блок страницы, который можно двигать и редактировать. Помечается двумя атрибутами:

```html
<section data-mpc-section="hero" data-mpc-name="Главный экран" id="{$id}">
  ...
</section>
```

- `data-mpc-section` — машинное имя секции (ключ конфигурации);
- `data-mpc-name` — человекочитаемое имя, которое увидит менеджер в админке;
- `id="{$id}"` — плейсхолдер идентификатора секции. Ставьте его, если секция может встретиться на странице несколько раз, чтобы у блоков были уникальные `id`.

У секции есть несколько вариантов разметки: обычная, **статическая** (`data-mpc-static` — одинаковая на всех страницах, см. [Статические секции](static-sections)) и **копия** (`data-mpc-copy` — та же вёрстка со своим отдельным контентом, см. [Копирование секций](copy-sections)). Полный разбор всех вариантов с примерами «вход → выход» — в этих разделах.

## Поле

`data-mpc-field` помечает редактируемое значение. Откуда брать значение, компонент определяет **сам по элементу**:

- ссылка (`<a href>`) → значением станет `href`;
- картинка (`<img>`) → берутся `src`, `alt`, `title`;
- любой другой элемент → берётся его внутреннее содержимое.

Пример секции с заголовком, текстом, картинкой и кнопкой:

```html
<section data-mpc-section="hero" data-mpc-name="Главный экран" id="{$id}">
  <h1 data-mpc-field="title">Заголовок страницы</h1>
  <p  data-mpc-field="subtitle">Короткое описание</p>
  <img data-mpc-field="img" src="assets/img/hero.png" width="1200" height="500" alt="">
  <a  data-mpc-field="link" href="/contacts">Связаться</a>
</section>
```

После нарезки:

- в админке появится секция «Главный экран» с полями `title`, `subtitle`, `img`, `link`;
- в файле секции на месте значений будут плейсхолдеры `{$title}`, `{$subtitle}`, `{$img}`, `{$link}`.

Если у картинки заданы `width`/`height`, к ней может подключиться обрезка под нужный размер — но только когда в системных настройках указан сниппет-обрезчик; если настройка пустая, картинка выводится как есть. Ленивая загрузка тоже включается через системные настройки. Какие именно это настройки и как ими управлять — в разделе [Работа с медиа](media).

## Списки повторяющихся блоков

Если блок повторяется (карточки, слайды, пункты меню) — это список. Контейнер помечается как поле, каждый повтор — атрибутом `data-mpc-item`, а поля **внутри** повтора получают индекс уровня вложенности (`-1`, `-2`, …):

```html
<div class="cards" data-mpc-field="cards">
  <div class="card" data-mpc-item>
    <img data-mpc-field-1="img" src="assets/img/1.png" alt="{$item1.title}">
    <h3 data-mpc-field-1="title">Первая карточка</h3>
    <p  data-mpc-field-1="text">Текст первой карточки</p>
  </div>
  <div class="card" data-mpc-item>
    <img data-mpc-field-1="img" src="assets/img/2.png" alt="{$item1.title}">
    <h3 data-mpc-field-1="title">Вторая карточка</h3>
    <p  data-mpc-field-1="text">Текст второй карточки</p>
  </div>
</div>
```

Что здесь важно:

- **вёрстка берётся только из первого** `data-mpc-item`, а контент — из всех (в админку попадут обе карточки);
- поля внутри списка адресуются через переменную элемента — например `$item1` для первого уровня (поэтому в `alt` уже стоит `{$item1.title}`);
- индекс растёт с вложенностью: список внутри списка → его повторы помечаются `data-mpc-item-1`, а поля в них — `data-mpc-field-2`, и так далее.

После нарезки контейнер превратится в MIGX-грид в админке (одна строка = одна карточка), а в файле секции компонент сам обернёт блок в цикл `{foreach}` и подставит плейсхолдеры — писать цикл руками не нужно.

## Сниппеты и чанки

**Вызов сниппета** — `data-mpc-snippet="ИмяСниппета|пресет"`. Элемент заменяется вызовом сниппета, параметры берутся из пресета по ключу (см. [Работа со сниппетами](snippets)):

```html
<ul data-mpc-snippet="pdoMenu|main_menu">...</ul>
```

После нарезки на месте элемента будет вызов сниппета с параметрами из пресета `main_menu`.

**Вырезание в чанк** — `data-mpc-chunk="путь/к/chunk.tpl"`. Содержимое элемента вырезается в отдельный чанк. Работает и само по себе, и **внутри вызова сниппета** (так размечают шаблон строки или обёртки списка):

```html
<li data-mpc-chunk="chunks/menuitem.tpl">
  <a href="{$uri}">{$menutitle}</a>
</li>
```

Как подключить вырезанный чанк обратно, задают парные атрибуты:

- `data-mpc-include` → `{include "file:chunks/promo.tpl"}`:
  ```html
  <div data-mpc-chunk="chunks/promo.tpl" data-mpc-include>
    <h2>Промо информация</h2>
  </div>
  ```
- `data-mpc-parse` → вызов `parseChunk` с параметрами из значения атрибута:
  ```html
  <div data-mpc-chunk="chunks/promo_2.tpl" data-mpc-parse="[$id => 1]">
     <h2>Промо информация из ресурса ID={$id}</h2>
  </div>
  ```
  превратится в `{$_modx->parseChunk("@FILE chunks/promo.tpl", [$id => 1])}`.

Подробно — в разделе [Работа с чанками](chunks).

**Удаление служебной разметки** — `data-mpc-remove`. Удаляет элемент из вёрстки после обработки. Нужен для служебных данных, которые участвуют в обработке, но не должны попасть в результат, — например, блока с инлайновыми стилями:

```html
<div data-mpc-field="section_styles" data-mpc-remove>
  .hero { padding: 80px 0 }
</div>
```

Содержимое уйдёт в поле `section_styles` (чтобы, например, подключить стили отдельно), а сам служебный `<div>` из вёрстки исчезнет.

**Снятие обёртки** — `data-mpc-unwrap`. В результат попадёт только содержимое элемента, без самого тега-обёртки. Пригодится, когда в значение одного поля нужно положить **сразу несколько тегов**:

```html
<div data-mpc-field="advantages" data-mpc-unwrap>
  <h3>Почему мы</h3>
  <ul><li>Быстро</li><li>Надёжно</li></ul>
</div>
```

В поле `advantages` попадут оба тега (`<h3>` и `<ul>`), а временный `<div>` в результат не войдёт — на его месте останется только `{$advantages}`, и вёрстка не изменится.

## Общие атрибуты полей

Эти атрибуты можно добавить к любому полю — обычному (`data-mpc-field`), полю ресурса (`data-mpc-rfield`) или TV (`data-mpc-tv`).

**`data-mpc-if`** — выводить поле только при выполнении условия. В значении — условие на Fenom; если оставить значение пустым, условием станет само значение поля (выводится, когда непустое):

```html
<a data-mpc-field="link" data-mpc-if="$link" href="/contacts">Связаться</a>
```

Здесь кнопка попадёт в результат, только если поле `link` заполнено. Условие можно задать и явно — например `data-mpc-if="$price > 0"`. Тот же атрибут работает и на секциях, и на вызовах сниппетов.

## Что ещё можно разметить

Кроме полей и сниппетов размечают и другие сущности:

- **`data-mpc-rfield`** — записать значение в нативное поле ресурса (`pagetitle`, `longtitle`, `content` и т.д.). На рендере — `{$resource.<поле>}`:
  ```html
  <h1 data-mpc-rfield="pagetitle">Заголовок страницы</h1>
  ```
- **`data-mpc-tv`** — записать значение в TV ресурса; если такого TV ещё нет, компонент создаст и синхронизирует его по разметке (см. [Создание и обновление элементов](elements)):
  ```html
  <span data-mpc-tv="subtitle">Подзаголовок</span>
  ```
- **`data-mpc-info`** — записать значение в системную или контекстную настройку MODX (контекст указывают в `data-mpc-ctx`); на рендере выводится значение настройки. Критичные настройки защищены от записи:
  ```html
  <span data-mpc-info="site_name" data-mpc-ctx="web">Мой сайт</span>
  ```
- **`data-mpc-contact`** — блок контактов (телефоны, почты, адреса, соцсети); поля контакта помечаются `data-mpc-cfield` (см. [Работа с контактами](contacts)):
  ```html
  <div data-mpc-contact="phone">
    <span data-mpc-cfield="value">+7 900 000-00-00</span>
    <span data-mpc-cfield="caption">Телефон</span>
  </div>
  ```

После того как шаблон размечен, остаётся запустить нарезку — и контент окажется в админке. О том, как создаются и обновляются элементы сайта, читайте в разделе [Создание и обновление элементов](elements).
